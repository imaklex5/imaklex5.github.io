初探hello world

Orzbot
2022-11-08
笔记, 编译

试着想一想，在我们使用集成开发环境(IDE)的时候是如何运行一个 hello world 程序的，在我们编码完成之后，只需要点击一下运行按钮，一句最经典的“Hello World”就会出现在控制台中。在这个过程中，编译器为我们提供了默认的编译、链接参数，才使得编译的过成变得如此简单。也就是这样我们往往会忽视强大的功能背后细节——运行键的背后隐藏了什么？

对于简单的 hello world 程序：

在 linux 下使用 gcc 进行编译和运行只需要一行命令：gcc hello.c && run a.out就可以得到想要的输出。

在这一条命令背后，gcc 调用了其他的程序来完成了编译的四个步骤：预处理、编译、汇编和链接。也就是说，gcc 其实是一堆后台程序的包装，它会根据不同的参数要求去调用预编译编译程序、汇编器、连接器

预编译(Preprocessing)

预处理过程由预编译器(cpp)来完成，其输入是后缀为.c/.cpp/.cxx/.hpp 的字符文件，处理对象是文件中’#’开头的部分，输出则为后缀为.i/.ii 的字符文件，这一步相当于如下命令:

预编译的过程主要处理源代码中的预编译指令，其规则如下：

  • 将所有宏定义#deifne删除，并展开所有宏定义。也就是说我们可以通过查看预编译文件来判断我们的宏是否正确展开了
  • 处理所有的条件预编译指令，如#if,#endif,#elif,ifdef等
  • 处理预编译指令#include ，将所有引入的头文件插入到指令对应的位置。这个过程是递归进行的，也就是说如果引入的文件中也包含#include指令，则也会对文件进行引入
  • 删除所有注释
  • 添加行号和文件名标识，用于 debug 和异常
  • 保留所有#pargma指令，编译器会用到这些指令

经过这个过程，最终得到的.i/.ii 文件将不会含有任何宏定义。

我们的源代码文件其实就是一个文本文件，现在是想，如果让你来看一个 c 的源代码是否正确，你会怎么做？首先，从最简单的开始，我们要知道我们需要对什么进行检查，然后就是需要对语法做检查，在此之后就是对程序的逻辑部分做检查了。大体看来，编译器的想法是和我们差不多的。

词法分析

这一步就是要列出我们需要检查的内容的清单，并形成一定结构。源代码会经由一个扫描器，该扫描器应用类似于有限自动机的算法来将我们的文本内容切分为一系列记号(关键字、标识符、字面量、特殊符号)并将其存放的符号表中

语法分析

在完成词法分析之后，语法分析器就会对符号表中的内容进行语法分析，最后会生成语法树，这是一棵以表达式为结点的树，整个过程采用上下文无关语法。在进行语法分析的同时，很多运算符的优先级和含义就被确定下来了，这个过程中程序中的表达式不合法的错误就会被检测出来，编译器就会爆出语法分析阶段的错误。

语义分析

这个过程由语义分析器来完成，虽然被称为语义分析，但是编译器只是完成表达式层面的分析，它并不了解语句的真实含义。其所能分析的只是静态语义(声明、类型匹配、类型转换)。经过此过程，整个语法树的表达式就会被标识上类型，如果有类型转换的需求，那么相应的地方就会被加上转换节点。

中间语言生成

上述三个过程是我们可以轻易想到的。但是，在现代编译器中往往还有这很多层次的优化，在源代码级别也会有一个优化过程，该过程实用的是源码优化器。直接对树进行优化是比较困难的，所以源码优化器通常会把语法树转换成中间代码(语法树的顺序表示，如：三地址码、P-代码等)，这样的代码很接近真实代码，但是它是和目标机器和运行环境无关的，也正是因为这个特点，中间代码使得表一起可以分为前端和后端。前端只负责生成与机器无关的中间代码，后端则负责将中间代码转换成目标机器代码。

代码生成与优化

这个过程就是编译器后端的内容，主要包括代码生成器、目标代码优化器。代码生成器将中间代码转换成目标机器代码，这个过程十分依赖于目标机器(字长、寄存器、整型、浮点型等)。目标代码优化器则负责对目标代码进行优化，如使用合适的寻址方式，位移替代乘法等。

链接(Linking)

链接则是一个十分难以理解的过程，该过程使用一个叫做链接器的程序(linker)，它接受一个或多个目标文件作为输入，经过链接之后，输出一个可执行文件，要生成一个 hello world 程序，在链接过程需要如下指令：

在前面的编译过程中我们已经看到了编译器其实是有摆烂的嫌疑的，问题来了，为啥他就可以放心大胆的摆烂？这就是链接器登场的时机了，链接器会解析程序中的未定义符号，重新计算各个目标的地址(重定位)，然后将目标文件中的占位符替换为符号的地址。

在汇编语言中，有一个很重要的概念：符号，一个符号用来表示一个地址，这个地址可以是变量起始地址，也可以是一个子程序的入口地址。随着软件规模的扩大，把几十万几百万行代码放在一个源文件中维护想想就可怕，于是我们就需要把程序划分成不同的功能模块来进行组织。问题来了：不同模块之间如何通信？

在 C/C++中最常见的有模块间的函数调用和模块之间的变量访问。怎么访问？我们肯定需要一个地址，所以这两种方式都可以归结为一种方式：符号引用，以符号引用为线索，将不同的模块拼接起来，这就是链接器所做的事，有点像拼积木对吧。

静态链接

链接的主要内容就是处理好各个模块之间相互引用的部分，使得各个模块之间可以正确的衔接通信，从原理上讲就是把一些指令对其它符号地址的引用进行修正。链接的过程主要包括：地址和空间分配、符号决议(也叫符号绑定、名称绑定、地址绑定)和重定位等。

最基本的静态链接过程就是：源文件经过编译生成目标文件，目标文件和库一起编译生成可执行文件，库其实是一组目标文件的包，最常见的库就是运行时库，他是支持程序运行的基本函数集合。在编译时，编译器会将无法找到的符号地址暂时搁置，等待最后链接器来对这些符号引用进行修正。也就是说，是链接器使得我们可以直接引用其他模块的函数和全局变量而不需要知道他们的地址。这个地址修正的过程就是重定位，每一个需要被修正的地方就是重定位入口。
